Problem Statement

Background

People consume media across many platforms—movies, TV shows, and video games—but it can be hard to keep track of what they want to watch or play, what they are currently enjoying, and what they have already completed. Existing tools are often limited to a single type of media and do not provide a centralized place to track progress, make notes, or see overall habits over time. This makes it difficult for users to organize their media experiences or reflect on what they enjoy most.

Problem Definition

There is a need for a unified system where users can manage all of their media in one place. Without such a system, media tracking is fragmented, progress is hard to measure, and users cannot easily plan what to watch or play next.

Proposed Solution

The MediaBacklog Web App will provide a single platform for users to manage their personal media backlog. In the minimum viable product (MVP), users can track movies and TV shows, including progress, personal ratings, and notes. In the future, support for video games will be added.

Key features of the platform include:
- A personal media backlog to track planned, in-progress, and completed media. 
- The ability to record ratings, reflections, and notes for each item. 
- Discovery of new media using external APIs. 
- A centralized, user-friendly interface that works for all supported media types.

The project also serves as a learning platform for enterprise Java development, demonstrating:
- User authentication and authorization 
- Persistence using Hibernate and JPA 
- API integration for media discovery 
- Layered application architecture 
- Logging and automated testing

This system will make it easier for users to organize, reflect on, and discover media while providing a solid foundation for expanding to additional media types in the future. with Hibernate and JPA, API integration, layered architecture, logging, and automated testing.
